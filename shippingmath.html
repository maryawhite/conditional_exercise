What does "use strict" do in JavaScript
In strict mode you can't use undeclared variables.
What's the benefit of using strict mode.
If you accidentally misspell a variable name, it will result in an error. Without strict mode, it would create a brand new global variable.
Other benefits of strict mode.
It won't allow you to duplicate parameters or use Octal numbers, and the "this" keyword behaves differently in strict mode, plus a few other benefits.

What is hoisting?
When you declare a variable using "var", the variable declaration is hoisted to the top of the code, which means you could use the var before it's declared, but JavaScript only hoists declarations, not initializations. W3 schools recommends that you always declare your variables at the beginning of the scope. Also note that "let" and "const" behave differently. Let and Const are hoisted to the top of the block, but not initialized. Trying to use a let variable before it is declared results in a Reference Error. Using const before it's declared is a syntax error and the code won't run.

What's the difference between an undeclared and undefined variable in JavaScript?
An undefined variable means a variable has been declared, but it doesn't have a value. An undeclared variable has not been declared and does not exist in the program. An undeclared variable will result in a Reference Error.

How would you convert a string to an integer in JavaScript?
To convert a string to an integer, you can use parseInt(). It will return NaN if no number is found. If spaces are included, it will return the number before the space and omit the rest. You can optionally include a radix. To convert from decimal to binary you could use parseInt("01001", 2); where the 2 stands for base 2 which is binary. This would return 9 because the binary number 01001 equals the decimal number 9.
